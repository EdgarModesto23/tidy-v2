# Impeccable — Tidy

## Design Context

### Users

**Inferred from the product (no README positioning yet):** People who want a **simple, private task list** in the browser—no accounts implied, local storage only. Typical contexts: quick capture during work, personal errands, lightweight planning. **Not yet confirmed:** age range, locale, or whether this is strictly personal vs. small-team shared later.

### Brand Personality

**Working hypothesis:** **Tidy** suggests order without fuss—**calm**, **direct**, and **trustworthy**. Voice in copy is short and human (“Nice work.”). Three-word personality: **clear · grounded · capable**.

**Emotional goals:** Reduce mental clutter; reward completion without cheerleading. Avoid anxiety (no red alarm states for normal actions).

### Aesthetic Direction

**Current execution (post-color pass):** **Warm, grounded dark UI** with **amber** as the primary action and wayfinding accent (distinct from generic cyan/mint-on-dark). **Sage-toned success** for completion (checkboxes, done states). **Rose/coral** for destructive actions. Neutrals are **warm-tinted** (not pure gray slabs). Background depth uses **subtle warm radial light** plus a very soft secondary glow—intentional atmosphere, not a purple-blue “AI gradient.”

**Theme:** Dark-first for now; light mode not implemented.

**Anti-references (explicit):** Neon mint/cyan as the only “brand” color on black; purple–blue mesh gradients; glassmorphism for decoration; monospace for ordinary UI copy.

### Design Principles

1. **Color means something** — Amber = act / navigate; green family = completed; rose = destructive; muted warm grays = structure and secondary text.
2. **Restraint over spectacle** — One primary action per view; chips and filters stay visually quieter than “Add.”
3. **Warm neutrals** — Surfaces and borders use slight hue (OKLCH) so the UI feels cohesive and human, not terminal-gray.
4. **Accessible by default** — Focus states visible; never rely on color alone (checkbox + label + strikethrough for done).
5. **Honest empty states** — Copy orients the user to the next step, not just “nothing here.”
